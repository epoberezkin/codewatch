Executive Summary

The security audit of the expressjs project (express, body-parser, cors) identified 11 findings across 54 files analyzed. No critical vulnerabilities were found. One high-severity issue relates to prototype pollution in the query string parser. Three medium-severity findings involve insufficient input validation in body-parser content-type handling, potential ReDoS in route parameter patterns, and missing security headers in default configuration. The overall security posture is good, with well-structured code and consistent error handling patterns throughout the codebase.

Findings

High Prototype Pollution via Query String Parser
express/lib/middleware/query.js:24-38

The query string parsing middleware uses qs library with default settings that allow nested object creation. An attacker can inject __proto__ properties through crafted query parameters, potentially modifying Object prototype and affecting application behavior globally.

CWE

CWE-1321: Improperly Controlled Modification of Object Prototype Attributes

CVSS

7.3 (High) - CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:L/I:L/A:L

Affected Code

// express/lib/middleware/query.js
module.exports = function query(options) {
  var opts = merge({}, options);
  var queryparse = qs.parse;

  // Default options do not set allowPrototypes: false
  return function query(req, res, next) {
    if (!req.query) {
      var val = parseUrl(req).query;
      req.query = queryparse(val, opts);  // Vulnerable line
    }
    next();
  };
};

Recommendation

Set allowPrototypes: false explicitly in the qs parser options. Consider adding parameterLimit and depth restrictions as defense-in-depth measures.

Medium Insufficient Content-Type Validation in Body Parser
body-parser/lib/types/json.js:41-56

The JSON body parser accepts content types matching */json without strict validation. A maliciously crafted Content-Type header could bypass security middleware that relies on content-type filtering.

Medium Potential ReDoS in Route Parameter Pattern
express/lib/router/layer.js:45-62

Complex route parameter patterns with nested quantifiers may cause catastrophic backtracking when matching against specially crafted URL paths, leading to denial-of-service conditions.

Low Missing X-Content-Type-Options Header
express/lib/response.js:118

The default response does not include the X-Content-Type-Options: nosniff header, which can allow MIME type sniffing in older browsers.

Info CORS Wildcard Origin Allowed by Default
cors/lib/index.js:12-28

When initialized without an explicit origin configuration, the cors middleware allows all origins (*). While this is documented behavior, applications relying on default settings may unintentionally expose APIs to cross-origin requests.
